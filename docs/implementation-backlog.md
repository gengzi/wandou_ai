# Wandou AI 待实现计划清单

> 生成时间：2026-05-17  
> 范围：基于当前仓库代码、`PROJECT_NOTES.md`、`docs/design-language.md`、前后端接口和 Docker 配置的实现缺口梳理。

## 当前已具备的基础

- 已有 React + Vite 前端、Node/Express 托管与代理、Spring Boot 后端。
- 已有 Sa-Token 登录、角色/权限、BCrypt 密码、Flyway 数据库迁移、PostgreSQL 持久化。
- 已有项目、画布、会话、任务、素材、模型配置、用量统计、Agent Run、SSE 事件、公开回放等核心接口。
- 已有 MinIO 对象存储接入、素材上传/外部导入、部分文件解析、模型配置管理。
- 已有多 Agent 视频工作流骨架：剧本、角色、分镜、关键帧、视频片段、音频节点、最终节点、确认/打断/恢复/取消。
- 已有文本、图片、视频模型的基本适配边界：OpenAI 兼容文本/生图、Pollinations、生图任务接口、QWave/Qingyun 视频任务接口，以及 mock/fallback 路径。

## P0：上线前必须补齐

- [ ] **修正文档与真实实现不一致**
  - `PROJECT_NOTES.md` 仍描述“后端当前使用内存仓库”，但当前代码已经使用 JPA、PostgreSQL、Flyway。
  - 补充当前真实接口、存储、模型配置、对象存储、Agent Run、回放、用量统计说明。
  - 给出本地开发、Docker Compose、真实模型接入、常见错误排查的最新说明。

- [ ] **项目与数据权限隔离**
  - 为项目、画布、会话、任务、素材、Agent Run 加入 `owner_user_id` 或租户/工作区字段。
  - 查询接口按当前登录用户、角色和项目成员关系过滤，避免所有登录用户看到全量项目。
  - 引入项目成员表、协作角色、邀请接受流程，而不只是管理员直接创建用户。
  - 公开回放需要显式分享开关、访问令牌、过期时间和撤销能力。

- [ ] **模型密钥安全**
  - 模型配置里的 API Key 需要加密存储，避免以明文字段持久化。
  - 支持密钥轮换、删除后不可恢复、只展示掩码预览。
  - 后端日志和错误响应不得泄露 provider key、签名 URL、完整鉴权头。

- [ ] **真实生成链路闭环**
  - 最终成片节点目前聚合 clips，但还没有真正把多个视频片段、音频、字幕合成为一个长视频。
  - 接入后端合成器，例如 FFmpeg 服务/任务队列，输出 durable final asset。
  - 音频节点需要真实音频/配乐/字幕生成能力，或者在 UI 中明确显示为未启用能力。
  - 视频任务应支持 provider 回调或持久轮询状态，避免只依赖当前 JVM 内存中的执行器。

- [ ] **Agent Run 持久化执行**
  - 当前活动运行状态仍部分依赖内存 map 和本地线程池，服务重启后无法继续运行。
  - 引入持久状态机、任务队列、重试策略、幂等 step id、provider job 恢复。
  - 将确认、打断、恢复、取消语义落到可恢复的运行状态，而不只是唤醒内存对象。

- [ ] **核心 UI 行为全部接真实后端**
  - 逐个检查画布节点按钮，确保重新生成、图片转视频、变体、批量生成、下载、播放、删除都对应真实 API。
  - 对暂未实现能力隐藏或禁用，并给出简短中文原因。
  - 避免“看起来能点但没有效果”的控件，尤其是音频播放、最终成片导出、分享与高级筛选。

## P1：产品主流程增强

- [ ] **完整的创作工作流参数化**
  - 将比例、分辨率、时长、音频、多镜头、模型选择写入节点参数、任务记录和素材 metadata。
  - 节点重跑时默认复用上次参数和引用素材。
  - 增加工作流级 preset，例如故事短片、商品广告、角色设定、分镜包、贴纸/周边。

- [ ] **角色与参考图一致性**
  - 支持多角色抽取、角色卡、角色参考图版本、角色锁定。
  - 下游关键帧、视频、衍生品默认继承角色与参考图。
  - 提供“替换参考图后重跑下游节点”的真实操作。

- [ ] **素材库升级为项目记忆**
  - 加入素材版本、来源链路、引用关系、批量操作、标签、收藏、归档。
  - 上传脚本、分镜文档、图片、视频、音频后形成可检索上下文。
  - 为视频生成缩略图、时长、尺寸、编码信息；为图片生成尺寸、主色、来源提示词。
  - 下载应走后端鉴权或短期签名 URL，避免长期公开对象 URL。

- [ ] **公开回放与协作**
  - 分享链接需要访问控制、可撤销 token、只读回放视图、隐藏敏感模型配置和内部错误。
  - 协作者可以加入项目、评论节点、引用节点输出到对话。
  - 增加项目活动流：谁在什么时候启动了 Run、编辑了节点、创建了素材。

- [ ] **用量与计费更准确**
  - 将 credits 从粗略请求计数升级为 provider 维度的真实计费模型。
  - 记录输入/输出 token、图片张数、视频秒数、分辨率、失败退款/不计费策略。
  - 增加用户/项目预算、用量告警、管理员汇总报表。

- [ ] **模型配置体验**
  - 保存前校验模型配置是否可用，提供测试调用按钮。
  - 按能力展示 provider 支持参数，例如图生视频、首尾帧、音频、分辨率、时长范围。
  - 区分个人模型配置、工作区共享模型配置、系统默认模型配置。

## P2：工程质量与运维

- [ ] **测试体系补齐**
  - 后端补充项目权限隔离、素材对象鉴权、Agent Run 状态机、provider 失败恢复、公开回放安全测试。
  - 前端补充关键页面组件测试和 API 错误态测试。
  - 增加 Playwright E2E：登录、创建项目、上传参考图、启动 Agent Run、确认、生成素材、回放。
  - 增加真实 provider 的契约测试，默认用 mock 录制/回放，避免 CI 依赖外部服务。

- [ ] **CI/CD 与发布**
  - 增加 GitHub Actions 或等价 CI：前端类型检查、构建、后端测试、镜像构建。
  - 加入数据库迁移校验、Docker Compose smoke test、基础安全扫描。
  - 为 staging/production 分离配置，明确密钥、域名、CORS、对象存储、数据库备份。

- [ ] **观测与审计**
  - 增加结构化日志、trace id、run id、project id、provider request id 贯穿前后端。
  - 暴露关键指标：运行成功率、平均耗时、provider 错误率、队列长度、对象存储错误。
  - 增加审计日志：登录、登出、用户邀请、权限变化、模型密钥变更、公开分享变更。

- [ ] **性能与稳定性**
  - 画布节点多时需要虚拟化/分区加载，避免一次性渲染所有历史节点。
  - SSE 断线重连要基于事件 id 恢复，避免重复消息或漏消息。
  - 素材列表需要服务端分页、搜索索引、懒加载缩略图。
  - 大文件上传需要大小限制、断点/分片上传策略、异步解析队列。

- [ ] **国际化与中文默认体验**
  - 当前 i18n 主要覆盖登录和首页，工作台、素材、用户、用量、模型设置仍有大量硬编码文案。
  - 默认中文 UI 需要完整落到翻译表，英文作为可选语言。
  - Provider 名称、模型名可以保留英文；操作、错误、提示、空状态应优先中文。

## P3：后续产品能力

- [ ] **高级视频生产能力**
  - 时间线编辑、镜头排序、剪辑替换、转场、字幕样式、配音音轨。
  - 首尾帧图生视频、角色动作变体、局部重绘、镜头续写。
  - 多 provider 路由：失败自动切换、成本/速度/质量策略、人工确认降级。

- [ ] **衍生品与商业交付**
  - T 恤、贴纸、海报、商品 mockup 当前已有基础路径，需要扩展到可选尺寸、工艺、出血线、印刷包规范。
  - 支持 3D 模型概念、可打印模型、角色表情包、姿态包。
  - 交付包需要包含素材、授权/来源、提示词、模型配置摘要、生成时间和版本。

- [ ] **企业集成**
  - SSO/OIDC：Keycloak、Microsoft Entra ID、Auth0、Okta 等。
  - 组织/部门/项目空间、细粒度角色、审批流。
  - Webhook/API：外部系统创建项目、监听生成完成、拉取资产。

## 推荐实施顺序

1. 先做 P0 的数据权限、密钥安全、最终成片合成、Agent Run 持久状态机。
2. 同步修正文档和 smoke test，让当前能力可被稳定复现。
3. 再做 P1 的素材库、模型配置体验、用量计费和协作。
4. 最后推进 P2/P3 的工程化、观测、企业集成和高级视频编辑能力。

## 完成定义

- 每个可见 UI 操作都有真实后端行为、错误态和权限校验。
- 所有生成输出都沉淀为 durable asset，并能从项目、画布、会话、任务和回放中追踪。
- 服务重启后，历史项目、运行记录、素材、事件和可恢复任务仍可查询和继续处理。
- 默认中文体验完整，敏感信息不泄露，项目数据按用户/工作区隔离。
- 本地、CI、Docker Compose 至少覆盖一条完整创作链路的自动化验证。
